## Decisions Made

- **kore-llm module**: The plan's discretion note said kore-core must have zero external deps (D-15). LLM provider SDKs (Anthropic, OpenAI, LangChain4j) require JVM HTTP clients and provider-specific types — creating `kore-llm` keeps kore-core clean while housing all four adapters. Correct hexagonal architecture interpretation.
- **kore-test api() declarations**: Using `api()` instead of `implementation()` for MockK, Kotest, coroutines-test ensures these test utilities propagate transitively to kore-llm and kore-mcp test scopes without each subproject having to re-declare them.
