2. Update KoreAutoConfiguration.kt DashboardAutoConfiguration inner class to correctly create DashboardServer:
   - Replace the reflection-based dashboardServer() bean with a direct instantiation now that kore-dashboard is an implementation dep:
     ```kotlin
     @Configuration(proxyBeanMethods = false)
     @ConditionalOnClass(name = ["dev.unityinflow.kore.dashboard.DashboardServer"])
     @ConditionalOnProperty(prefix = "kore.dashboard", name = ["enabled"], havingValue = "true", matchIfMissing = true)
     inner class DashboardAutoConfiguration {
         @Bean
         fun dashboardServer(
             eventBus: EventBus,
             auditLog: ObjectProvider<AuditLog>,
             properties: KoreProperties,
         ): DashboardServer {
             val dashProps = properties.dashboard
             return DashboardServer(
                 eventBus = eventBus,
                 auditLog = auditLog.ifAvailable,
                 config = DashboardConfig(
                     port = dashProps.port,
                     path = dashProps.path,
                     enabled = dashProps.enabled,
                 ),
             )
         }
     }
     ```
   - Import: `import dev.unityinflow.kore.dashboard.DashboardServer`, `import dev.unityinflow.kore.dashboard.DashboardConfig`
   - Import: `import org.springframework.beans.factory.ObjectProvider`
   - Note: ObjectProvider<AuditLog> allows optional injection without crashing when AuditLog bean is absent (though in practice InMemoryAuditLog is always registered by KoreAutoConfiguration, so it will be present). Use `ObjectProvider` for cleaner optional semantics.
