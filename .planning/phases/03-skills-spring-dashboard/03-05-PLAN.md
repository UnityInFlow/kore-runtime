<objective>
Close the two high-severity gaps from 03-VERIFICATION.md and 03-REVIEW.md that block Phase 3 completion:

**HI-01 — DashboardServer cannot restart.** `DashboardServer.scope` is a `private val` initialized once at class construction (DashboardServer.kt:82). `stop()` cancels it (line 106); a subsequent `start()` then launches the observer on a cancelled scope, producing a silent empty active-agents view. Spring SmartLifecycle contract (and any host that restarts the bean during context refresh) is violated. Fix: hold `scope` and `observer` behind `AtomicReference` and recreate them on each `start()` when the current scope is null or inactive.

**HI-02 — Degraded-mode notice unreachable via Spring wiring.** `KoreAutoConfiguration.inMemoryAuditLog()` unconditionally supplies `InMemoryAuditLog` as the default `AuditLog` bean (KoreAutoConfiguration.kt:62-63). In the zero-config hero-demo Spring path, `DashboardDataService` therefore receives a non-null `InMemoryAuditLog` and `hasStorage()` returns `true`, so the `/kore/fragments/recent-runs` and `/kore/fragments/cost-summary` endpoints render empty tables instead of the UI-SPEC.md degraded copy. The "History unavailable — kore-storage not configured" notice is unreachable on the default Spring wiring path. Fix: add `val isPersistent: Boolean = false` to the `AuditLog` port, override to `true` in `PostgresAuditLogAdapter`, and make `DashboardDataService.hasStorage()` return `auditLog?.isPersistent == true`.

Purpose: Unblock Phase 3 verification gate with minimal, mechanical fixes. Each fix is TDD-ordered (red test first, then code). No scope beyond the two findings — no opportunistic refactors.
Output: Updated `AuditLog` port + three impls + `DashboardDataService` + `DashboardServer`; three new regression tests (one per task + integration test); all existing Phase 3 tests still passing.
</objective>
