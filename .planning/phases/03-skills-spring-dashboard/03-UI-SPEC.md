# Phase 3 — UI Design Contract

> Visual and interaction contract for Phase 3: Skills, Spring & Dashboard.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Scope: Single HTMX dashboard page at `/kore`. Server-rendered via kotlinx.html DSL.
> No JavaScript framework, no CSS framework, no build step.
> Tech: Ktor 3.2 embedded server + ktor-server-htmx + ktor-server-html-builder.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none | CONTEXT.md D-32: no JS framework, no build step |
| Preset | not applicable | No shadcn — Kotlin/Ktor stack, not React/Next.js |
| Component library | none | kotlinx.html DSL generates all HTML server-side |
| Icon library | none | Status conveyed via text symbols and HTML entities (no icon SVG library required in v0.1) |

## Spacing Scale

Declared values (must be multiples of 4). Expressed as inline CSS values in kotlinx.html attributes since there is no CSS preprocessor or utility class system.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-adjacent gaps, badge padding, table cell vertical padding |
| sm | 8px | Status pill padding, compact row spacing |
| md | 16px | Card internal padding, form field spacing, nav bar padding |
| lg | 24px | Section heading bottom margin, fragment container padding |
| xl | 32px | Gap between the three dashboard fragments |
| 2xl | 48px | Page top padding |
| 3xl | 64px | Not used in v0.1 (single-page, no hero sections) |

Exceptions:
- Table row height: 40px minimum (touch-accessible, platform engineer persona uses on large monitors)
- Page max-width: 1200px centered with `margin: 0 auto`

---

## Typography

No external font load. All sizes in `rem` with base 16px browser default. Weights use numeric values for system font stack compatibility.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (0.875rem) | 400 | 1.5 | Table cell content, status labels, fragment body text |
| Label | 12px (0.75rem) | 600 | 1.4 | Table column headers, section labels, badge text |
| Heading | 18px (1.125rem) | 600 | 1.2 | Fragment section titles ("Active Agents", "Recent Runs", "Cost Summary") |
| Display | 24px (1.5rem) | 600 | 1.2 | Page title "kore dashboard" in the top nav bar |

Font family (applied once on `body`):
```
font-family: system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

Monospace (used for agent IDs, run IDs, model names):
```
font-family: ui-monospace, "SF Mono", "Fira Code", Consolas, monospace;
font-size: 13px;
```

## Color

Minimal palette. Dark mode is not required for v0.1. Colors use hex literals embedded in kotlinx.html style attributes or a single embedded `<style>` block at page head.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#FFFFFF` | Page background, card/fragment background |
| Secondary (30%) | `#F5F5F5` | Table row alternate stripe, nav bar background, fragment outer wrapper |
| Accent (10%) | `#2563EB` | Active agent indicator dot, "running" status badge border, primary nav link |
| Success | `#16A34A` | "success" result badge text and border |
| Warning | `#D97706` | "budget_exceeded" result badge text and border |
| Destructive | `#DC2626` | "error" / "tool_error" / "llm_error" result badge text and border |
| Border | `#E5E7EB` | Table borders, card outline, fragment separator |
| Text primary | `#111827` | All body text, headings, labels |
| Text secondary | `#6B7280` | Timestamps, secondary metadata, empty state body text |

Accent (`#2563EB`) reserved for:
- The animated pulse dot next to agents with status `RUNNING`
- The border on the "running" status badge
- The nav bar link to the current page (`/kore`)

All other interactive elements (no user-triggered navigation exists in v0.1 — read-only dashboard).

```

│                                         │
│  ┌─────────────────────────────────┐    │
│  │ FRAGMENT: active-agents         │    │  hx-get="/kore/fragments/active-agents"
│  │ hx-trigger="every 2s"           │    │  hx-swap="innerHTML"
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ FRAGMENT: recent-runs           │    │  hx-get="/kore/fragments/recent-runs"
│  │ hx-trigger="every 5s"           │    │  hx-swap="innerHTML"
│  └─────────────────────────────────┘    │
│                                         │
│  ┌─────────────────────────────────┐    │
│  │ FRAGMENT: cost-summary          │    │  hx-get="/kore/fragments/cost-summary"
│  │ hx-trigger="every 10s"          │    │  hx-swap="innerHTML"
│  └─────────────────────────────────┘    │
└─────────────────────────────────────────┘
```

### Fragment: Active Agents (`/kore/fragments/active-agents`)

Displays `ConcurrentHashMap<AgentId, AgentState>` from `EventBusDashboardObserver`.

Table columns: `Agent Name` | `Status` | `Current Task (truncated 60 chars)` | `Tokens Used` | `Running Since`

Status badge values and colors:
- `RUNNING` — accent blue border (#2563EB), blue text, animated pulse dot
- `IDLE` — border #E5E7EB, text secondary #6B7280
- `COMPLETED` — success green border (#16A34A), green text
- `ERROR` — destructive red border (#DC2626), red text

### Fragment: Recent Runs (`/kore/fragments/recent-runs`)

Queries PostgreSQL via `AuditLog.queryRecentRuns()`. Shows last 20 runs.

Table columns: `Agent Name` | `Result` | `Input Tokens` | `Output Tokens` | `Duration` | `Completed At`

Result badge values:
- `SUCCESS` — success green
- `BUDGET_EXCEEDED` — warning amber
- `TOOL_ERROR` — destructive red
- `LLM_ERROR` — destructive red
- `CANCELLED` — secondary gray

Degraded mode (kore-storage absent): show in-memory event data from EventBus with note "History unavailable — kore-storage not configured."

### Fragment: Cost Summary (`/kore/fragments/cost-summary`)

Queries PostgreSQL via `AuditLog.queryCostSummary()`. Groups by agent name.

Table columns: `Agent Name` | `Total Runs` | `Total Input Tokens` | `Total Output Tokens` | `Total Tokens`

Footer row: bold totals across all agents.

Degraded mode: "Cost history unavailable — kore-storage not configured."

## HTMX Interaction Contract

Source: CONTEXT.md D-30.

| Fragment endpoint | hx-trigger | hx-swap | hx-indicator |
|---|---|---|---|
| `/kore/fragments/active-agents` | `every 2s` | `innerHTML` | none (silent background refresh) |
| `/kore/fragments/recent-runs` | `every 5s` | `innerHTML` | none |
| `/kore/fragments/cost-summary` | `every 10s` | `innerHTML` | none |

No `hx-push-url`. No `hx-prompt`. No `hx-confirm`. No `hx-post`. Dashboard is read-only in v0.1.

HTMX CDN script tag: `https://unpkg.com/htmx.org@1.9.x/dist/htmx.min.js` embedded inline in the page `<head>` via kotlinx.html. No local asset serving required (Ktor serves only the dynamic HTML fragments, not static JS files).

## States and Interaction Map

| State | Trigger | Visual Treatment |
|-------|---------|-----------------|
| No agents running | EventBus empty | Active agents fragment shows empty state |
| No run history | PostgreSQL empty or absent | Recent runs fragment shows empty state |
| No cost data | PostgreSQL empty or absent | Cost summary fragment shows empty state |
| LLM error result | AgentResult.LLMError in DB | Row shows destructive red "LLM_ERROR" badge |
| Budget exceeded | AgentResult.BudgetExceeded | Row shows warning amber "BUDGET_EXCEEDED" badge |
| Degraded mode | kore-storage missing from classpath | Each historical fragment shows degraded notice |
| Server error on fragment | Ktor returns 500 | Fragment div retains its previous content (HTMX default — no swap on error) |

## Component Inventory

All components are server-rendered kotlinx.html functions in `kore-dashboard`. Not a component library — these are Kotlin helper functions producing HTML.

| Component | Kotlin Function | HTML Output |
|-----------|----------------|-------------|
| Page shell | `fun HTML.koreDashboardPage(content: BODY.() -> Unit)` | Full HTML document with `<head>` (meta, style, HTMX script) and `<body>` |
| Nav bar | `fun BODY.navBar(version: String)` | `<nav>` with page title and version |
| Fragment container | `fun BODY.fragmentContainer(id: String, pollSeconds: Int, content: DIV.() -> Unit)` | `<div hx-get="..." hx-trigger="every Ns" hx-swap="innerHTML">` |
| Section heading | `fun FlowContent.sectionHeading(title: String)` | `<h2>` with heading typography |
| Data table | `fun FlowContent.dataTable(headers: List<String>, rows: List<List<String>>)` | `<table>` with thead/tbody |
| Status badge | `fun FlowContent.statusBadge(status: AgentStatus)` | `<span>` with inline style for border and text color |
| Result badge | `fun FlowContent.resultBadge(result: AgentResultType)` | `<span>` with inline style |
| Empty state | `fun FlowContent.emptyState(heading: String, body: String)` | `<div>` centered with heading + body text |
| Degraded notice | `fun FlowContent.degradedNotice(message: String)` | `<div>` with secondary text and italic style |
| Monospace cell | `fun FlowContent.monoCell(value: String)` | `<td>` or `<span>` with monospace font style |

## CSS Delivery Strategy

No external CSS file, no CSS framework. Single `<style>` block embedded in `<head>` via kotlinx.html. All rules use class selectors (not inline styles) where possible for maintainability.

CSS classes to define in the embedded style block:

```
.kore-nav              — nav bar container
.kore-page             — main content wrapper, max-width 1200px
.kore-fragment         — fragment outer wrapper, card-like, border, bg
.kore-section-heading  — h2 styling
.kore-table            — table with borders, full width
.kore-table th         — label weight, bg #F5F5F5
.kore-table td         — body weight, border-bottom
.kore-table tr:hover   — hover bg #F9FAFB
.kore-badge            — base badge style (border-radius 4px, padding 2px 8px)
.kore-badge--running   — accent blue
.kore-badge--success   — success green
.kore-badge--warning   — warning amber
.kore-badge--error     — destructive red
.kore-badge--neutral   — secondary gray
.kore-empty            — empty state centered container
.kore-degraded         — degraded notice style
.kore-mono             — monospace text style
.kore-pulse            — CSS keyframe animation for RUNNING status dot
```

Keyframe for pulse dot (RUNNING indicator):
```css
@keyframes kore-pulse {
  0%, 100% { opacity: 1; }
  50% { opacity: 0.4; }
}
.kore-pulse { animation: kore-pulse 1.5s ease-in-out infinite; }
```

## Accessibility Contract

- All `<table>` elements have `<caption>` matching the section heading (screen reader navigation)
- Status badges use `aria-label` with full text (e.g., `aria-label="Status: Running"`)
- HTMX polling fragments are wrapped in `aria-live="polite"` region so screen readers announce updates
- Active agents fragment uses `aria-live="assertive"` (2s refresh, operator-critical)
- Color is never the sole differentiator — badges combine color + text label
- Minimum contrast ratio: 4.5:1 for all text on background (WCAG AA)
