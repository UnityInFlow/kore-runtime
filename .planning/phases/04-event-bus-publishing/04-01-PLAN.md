<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kore-core runtime classpath → consumer apps | Any class on kore-core's runtime classpath becomes a dependency of every consumer. Keeping it zero-external-dep (except coroutines) is a public-API surface decision. |
| @Serializable bytecode → external deserialization | Once kore-kafka / kore-rabbitmq ship, any party with broker access can emit JSON that attempts to deserialize into AgentEvent subclasses via the `type` discriminator. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | kore-core runtime classpath | mitigate | Task 2 verifies `./gradlew :kore-core:dependencies --configuration runtimeClasspath` does not contain `kotlinx-serialization-json`. compileOnly on `serialization-core` only. |
| T-04-02 | Information Disclosure | AgentEvent JSON wire format (Pattern 2) | accept | JSON is human-readable by design (broker UI debuggability is a product feature). If PII leakage becomes a concern, v0.1+ introduces field-level redaction. Not in scope for v0.0.1. |
| T-04-03 | Tampering | Polymorphic deserialization (@JsonClassDiscriminator) | mitigate | sealed class constraint means kotlinx.serialization only decodes known subclasses; unknown `type` values throw `SerializationException` (no arbitrary class loading). AgentEventSerializationTest asserts roundtrip for every known subclass — negative-test (unknown type rejected) deferred to kore-kafka/rabbit adapter tests. |
| T-04-05 | DoS (regression) | InProcessEventBus buffer config | mitigate | EventBusBackpressureTest asserts `testScheduler.currentTime == 0L` across 200 emits — any future regression that removes `extraBufferCapacity = 64, DROP_OLDEST` fails the test immediately (Pitfall 1 defense). |
