<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Self-hosted runner → Sonatype Central Portal API | Workflow execution machine holds signing key + portal credentials in memory during publish. |
| Git tag `v0.0.1` → released artifacts | Tag push is the trigger; anyone with write access to the repo can theoretically cut a release. |
| Sonatype staging bundle → Maven Central | Human-gated release button prevents accidental or forced publish. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-07 | EoP | GPG signing key leakage through CI logs (Pitfall 11) | mitigate | `--no-configuration-cache` on publish step ensures env vars are not serialized into cache files. GitHub Actions masks secret values in logs automatically. |
| T-04-09 | Repudiation | Which commit produced which artifact | mitigate | `actions/checkout@v4` with `fetch-depth: 0` ensures full history is present for GitHub-generated release notes. `softprops/action-gh-release@v2` with `generate_release_notes: true` creates a release linked to the tag commit. |
| T-04-10 | EoP | Compromised self-hosted runner publishes malicious artifact | accept (trust boundary documented) | `arc-runner-unityinflow` is org-owned; CLAUDE.md mandates self-hosted. Runner group access is controlled at GitHub org level. Runner compromise = org compromise, which is outside the scope of this plan. Human RC dry-run (Checkpoint 2) catches anomalous bundles via Sonatype portal inspection. PF-02 pre-flight audit verifies runner labels match expected hosts. |
| T-04-11 | DoS | Workflow accidentally re-runs on same tag | mitigate | Tag pushes are idempotent — Sonatype Central Portal rejects duplicate version uploads. Worst case: workflow fails at publish step; no harm done. |
| T-04-16 | Repudiation | No 2nd factor on the release button | accept | v0.0.1 is solo-dev ship. Future releases can add `environments:` with required reviewers if multi-person org. |
</threat_model>
